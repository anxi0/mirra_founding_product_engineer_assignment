# Mirra FPE 채용 과제

> 출처: Mirra_FPE_과제_Candid.pdf (2026.06.17)  
> 예상 소요: 36–48시간  
> 제출 형태: 저장소 1개 또는 문서 + 코드 링크

## 배경

Mirra는 소상공인의 콘텐츠 제작·예약 발행·관리·분석을 지원하는 마케팅 SaaS.  
Founding Product Engineer는 아래 **세 전선**을 동시에 다룬다. 모두 현재 Mirra가 실제로 풀고 있는 문제.

| 전선 | 키워드 | 핵심 |
|------|--------|------|
| **가 · 부채** | 기존 제품 리팩토링 | 필요할 때마다 기능을 덧댄 구조적 부채 → AI-native 하네스로 빠르고 안전하게 걷어내기 |
| **나 · 신규** | Agent-first 제품 빌드 | 발행·관리·분석 핵심은 유지, 흩어진 AI 툴은 정리 → "그냥 말하면 되는" 인터페이스로 재구성 |
| **다 · 플라이휠** | 쿼리 기반 데이터 분석 루프 | Agent 쿼리 = 1차 고객 인텐트 데이터 → 기존 퍼널 데이터와 병합해 분석 루프 구축 |

---

## Part A — 세 전선에 대한 관점

테스트 계정으로 실제 고객처럼 핵심 플로우를 끝까지 사용한 경험을 바탕으로 작성.  
분량: **방향당 ½–1페이지**. 길이보다 밀도.

### 가 · 부채
- 직접 사용하며 발견한 **구조적 문제 2–3개**와 근거
- 그 문제를 어떤 **하네스**로 걷어낼지, 핵심 접근 방식

### 나 · 신규
- Agent-first로 재설계한다면 SNS 발행·관리·분석을 어떻게 agent 인터페이스 뒤로 재배치할지
- **무엇을 남기고 무엇을 버릴지** 명시

### 다 · 플라이휠
- 사용자 쿼리를 데이터로 적재하고 기존 퍼널 데이터와 병합하기 위한 **이벤트 · 스키마 · 분석 루프** 설계

---

## Part B — 레버리지 선택과 첫 1주 계획

가/나/다 중 **하나를 선택**하고 아래 세 질문에 답한다.

1. **왜 이 방향인가** — "지금 Mirra에 가장 레버리지가 큰 한 수"라는 기준으로 논증. 본인 강점이 아닌 **회사 관점 우선순위**가 논거의 중심
2. **첫 1주 계획** — Day 단위 행동 계획
3. **1주 후 달라진 것** — 기준선 + 측정 가능한 지표

> 이 선택과 논거가 과제의 **가장 중요한 평가 지점**.

---

## Part C — 작동하는 PoC 1개

**Part C 없으면 과제 미완성.** Part A·B의 판단을 실제 결과물로 증명하는 단계.

Part B에서 선택한 방향 안에서 **가장 중요한 조각 하나**를 골라 실제로 구현.  
스코프는 작아도 되나, **반드시 작동**해야 하고 AI-native 하네스로 작은 루프를 돌린 결과물이어야 함.

### 방향별 예시 (그대로 따를 필요 없음)
- **가** — 실제/유사 코드 한 덩어리를 하네스로 리팩토링 + 회귀 검증
- **나** — 핵심 기능 1개를 감싸는 최소 agent 인터페이스 (자연어 입력 → 발행·관리 액션 데모)
- **다** — 쿼리 수집 → 구조화 적재 → 퍼널 병합 분석까지 이어지는 end-to-end 미니 파이프라인

### 제출물에 포함할 것
- [ ] 작동하는 코드
- [ ] 실행 및 검증 방법
- [ ] 검증 증거 (테스트·계측 결과·before/after 비교 등 실제 동작 근거)
- [ ] AI 사용 회고 — 어디에 썼고, 어디서는 믿지 않고 직접 검증했는지

## 평가 기준

| 축 | 보는 것 | 반영 방식 |
|----|---------|-----------|
| 코드 / 실물 (C) | PoC가 실제로 작동하는가. 검증이 충분한가 | **Pass / Fail 게이트** |
| 레버리지 판단 (B) | 왜 그 방향을 골랐는가. 회사 관점 우선순위 판단 | 순위 결정 · 최상위 가중 |
| 시야 (A) | 세 전선을 구조화할 수 있는가. 무엇을 버릴지 판단 | 순위 결정 |
| AI Native 깊이 (B·C) | 속도만이 아니라 안정성과 검증까지 설계했는가 | 순위 결정 |
| 커뮤니케이션 | 핫픽스와 구조 개선을 분리해 설명하는가 | 가점 |

**코드(C)는 합격선.** A·B가 좋아도 C가 약하면 통과 어려움.  
합격선 이후 우선순위는 **B의 선택과 논거**로 판단.

## 진행 규칙

- AI 도구 사용 **전면 허용** — 어떻게 쓰는지를 봄
- Part C에 AI 사용 회고 **필수**
- 질문 최대한 많이 할 것 (모르는 것을 드러내고 질문하는 능력도 평가 대상)
- 막히면 멈추지 말고 **무엇을 가정하고 넘어갔는지** 명시
- 완벽한 완성이 목표가 아님 — **무엇을 깊게 보고, 무엇을 의도적으로 버렸는지**가 핵심
